
# 1.基础命令
## 1.1 **free命令**
- `free` 是 Linux 系统中用于**查看内存使用情况**的核心命令，它能清晰展示物理内存（RAM）、交换空间（Swap）以及内核缓存的使用状态。
**常用参数**：

|参数|作用|示例|
|---|---|---|
|`-b`|以字节（Bytes）显示|`free -b`|
|`-k`|以 KB 显示（默认）|`free -k`|
|`-m`|以 MB 显示|`free -m`|
|`-g`|以 GB 显示|`free -g`|
|`-h`|人类可读格式（自动选单位）|`free -h` ← 推荐使用|
|`-s N`|每 N 秒刷新一次|`free -h -s 3`（每 3 秒更新）|
|`-t`|显示 Mem + Swap 总计行|`free -ht`|

## 1.2 df命令
`df`（disk free）是 Linux 中用于**查看文件系统磁盘空间使用情况**的核心命令。它能显示各挂载点的总容量、已用空间、可用空间及使用百分比，是日常运维和故障排查的必备工具。
**常用参数**：

|参数|作用|示例|
|---|---|---|
|`-h`|人类可读格式（自动转为 KB/MB/GT）|`df -h` ← 最常用|
|`-H`|以 1000 为单位（而非 1024）|`df -H`|
|`-T`|显示文件系统类型（如 ext4, xfs）|`df -Th`|
|`-i`|显示 inode 使用情况（而非块空间）|`df -i`|
|`--output=字段`|自定义输出列（如 `source,size,used`）|`df --output=source,size,used /`|
|`-x TYPE`|排除指定类型的文件系统|`df -x tmpfs -x devtmpfs`|

## 1.3uptime命令
`uptime` 是 Linux 系统中一个简洁但非常实用的命令，用于**显示系统运行时间、当前登录用户数以及系统负载情况**。它帮助管理员快速判断系统稳定性与当前资源压力。

## 1.4 ps命令
`ps`（Process Status）是 Linux 中用于**查看当前系统进程快照**的核心命令。它能显示进程的 PID、CPU/内存占用、运行状态、所属用户等关键信息，是排查性能问题、管理服务和监控系统的重要工具。
**常用参数：**

| 命令               | 作用               | 说明                                                      |
| ---------------- | ---------------- | ------------------------------------------------------- |
| `ps aux`         | 查看所有进程（BSD 风格）   | • `a`=其他用户进程  <br>• `u`=显示用户/内存/CPU  <br>• `x`=无控制终端的进程 |
| `ps -ef`         | 查看所有进程（POSIX 风格） | • `-e`=所有进程  <br>• `-f`=完整格式（含 PPID）                    |
| `ps -u username` | 查看指定用户的进程        | 如 `ps -u nginx`                                         |
| `ps -p PID`      | 查看指定 PID 的进程     | 如 `ps -p 1234`                                          |

## 1.5 **管道符 `|`**
管道符 | 是 Unix/Linux shell 中最核心的组合操作符之一，它的作用是：将前一个命令的输出（stdout）作为后一个命令的输入（stdin），从而实现多个命令的“流水线式”协作。
- 命令1 | 命令2 = 把 命令1 的标准输出（stdout）直接“喂给” 命令2 的标准输入（stdin），无需临时文件。
## 1.6 grep命令
grep（Global Regular Expression Print）是 Linux/Unix 系统中最强大的文本搜索工具，用于在文件或标准输入中查找匹配指定模式（字符串或正则表达式）的行，并输出结果。它是日志分析、代码检索、数据过滤等任务的核心命令。

**基本语法**：
```bash
grep [选项] '搜索模式' 文件...
```

**核心功能与常用选项：**

| 选项          | 作用            | 示例                               |
| ----------- | ------------- | -------------------------------- |
| `-i`        | 忽略大小写         | `grep -i 'ERROR' log.txt`        |
| `-v`        | 反向匹配（显示不匹配的行） | `grep -v '^#' config.conf`（过滤注释） |
| `-r` / `-R` | 递归搜索目录        | `grep -r 'TODO' ~/project/`      |
| `-n`        | 显示行号          | `grep -n 'main' script.py`       |
| `-l`        | 只输出匹配的文件名     | `grep -l '404' /var/log/*.log`   |
| `-c`        | 统计匹配行数        | `grep -c 'success' access.log`   |
| `-A N`      | 显示匹配行及后 N 行   | `grep -A 2 'Exception' app.log`  |
| `-B N`      | 显示匹配行及前 N 行   | `grep -B 1 'FAIL' test.log`      |
| `-C N`      | 显示匹配行及前后各 N 行 | `grep -C 3 'timeout' debug.log`  |

**正则表达式支持：**

| 模式      | 含义          | 示例                                       |
| ------- | ----------- | ---------------------------------------- |
| `.`     | 匹配任意单个字符    | `grep 'a.c' file` → 匹配 "abc", "a2c"      |
| `*`     | 匹配前一字符0次或多次 | `grep 'go*d' file` → "gd", "god", "good" |
| `^`     | 行首锚点        | `grep '^root' /etc/passwd`               |
| `$`     | 行尾锚点        | `grep 'bash$' /etc/passwd`               |
| `[abc]` | 字符集合        | `grep '[0-9]' data.txt` → 包含数字的行         |
| `\|`    | 或操作（需转义）    | `grep 'error\|warning' log.txt`          |

## 1.7 awk
`awk` 是 Linux/Unix 系统中**最强大的文本处理工具之一**，它不仅是一个命令，更是一门**完整的编程语言**（由 Alfred Aho、Peter Weinberger 和 Brian Kernighan 于 1970 年代在贝尔实验室开发，名字取自三人姓氏首字母）。  
`awk` 擅长处理**结构化文本**（如日志、CSV、表格数据），能按字段提取、计算、过滤、格式化输出，被誉为“**命令行里的 Excel**”。

与 grep（按行筛选）、sed（按行编辑）不同，awk 默认将每行按空白字符（空格、制表符）分割成多个字段（列），然后对字段进行操作。

**内置字段变量：**

|变量|含义|
|---|---|
|`$0`|整行内容|
|`$1`, `$2`, `$3`, ...|第1、2、3...个字段|
|`NF`|当前行的字段总数（Number of Fields）|
|`NR`|当前行的行号（Number of Records）|

基本语法：
```bash
awk [选项] '模式 { 动作 }' 文件...
```
- **模式**（pattern）：指定对哪些行执行动作（可省略，默认所有行）
- **动作**（action）：对匹配行执行的操作（可省略，默认 `print $0`）

常用选项：

|选项|作用|
|---|---|
|`-F "分隔符"`|指定输入字段分隔符（默认空白）|
|`-v var=value`|传递外部变量给 awk|
|`-f script.awk`|从文件读取 awk 脚本|
