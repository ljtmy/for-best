goroutine 负责并发执行，channel 负责通信，sync/context 负责同步与控制生命周期。

# goroutine
## 什么是goroutine？
**Goroutine** 是 Go 语言中实现并发编程的核心机制，它是一种轻量级的线程，由 Go 运行时（runtime）调度和管理。
## goroutine创建方法
### 1.启动普通函数（已创建的函数）
```go
package main

import (
	"fmt"
	"time"
)

func sayHello() {
	fmt.Println("hello goroutine")
}

func main() {
	go sayHello()

	time.Sleep(time.Second) //加休眠，因为主程序也是goroutine，main结束了程序就直接结束了，休眠一点时间给子goroutine执行
	fmt.Println("main end")
}
```
### 2.启动匿名函数（未创建）
```go
package main

import (
	"fmt"
	"time"
)

func main() {
	go func() {
		fmt.Println("匿名 goroutine 执行")
	}()

	time.Sleep(time.Second)
}
```
### 3. 带参数的goroutine
```go
package main

import (
	"fmt"
	"time"
)

func main() {
	name := "Go"

	go func(s string) {
		fmt.Println("hello", s)
	}(name)//推荐传入外部参数

	time.Sleep(time.Second)
}
```
# waitGroup
## 什么是waitGroup
`sync.WaitGroup` 是 Go 语言标准库中用于**同步**（Synchronization）的最常用工具。它的功能非常纯粹：**等待一组 Goroutine 全部执行完毕。**

你可以把它想象成一个“倒计时计数器”：

1. 你每启动一个任务，就让计数器 **+1**。
    
2. 每个任务完成后，就让计数器 **-1**。
    
3. 主程序（或某个协程）调用“等待”方法，如果计数器不为 0，它就**阻塞**在那里，直到计数器变成 0 为止。
## 基本用法
```go
package main  
  
import (  
	"fmt"  
	"sync"  
)  
  
func main() {  
	var wg sync.WaitGroup  
  
	wg.Add(3)  
  
	go func() {  
		defer wg.Done()  //使用 defer 确保即使函数出错，计数器也会减 1
		fmt.Println("任务1完成")  
	}()  
  
	go func() {  
		defer wg.Done()  
		fmt.Println("任务2完成")  
	}()  
  
	go func() {  
		defer wg.Done()  
		fmt.Println("任务3完成")  
	}()  
  
	wg.Wait()  
  
	fmt.Println("所有任务完成")  
}

```
核心方法有三个：

| 方法       | 作用                  |
| -------- | ------------------- |
| `Add(n)` | 增加等待的 goroutine 数量  |
| `Done()` | 表示一个 goroutine 执行完成 |
| `Wait()` | 阻塞等待所有 goroutine 完成 |
Done()也就是表示goroutine的数量减一
### 三大禁忌
- **禁忌：在 Goroutine 内部调用 `Add`**
    
    - **后果**：由于主协程运行极快，可能在子协程还没来得及 `Add(1)` 之前，主协程就已经执行到 `Wait()` 并发现计数器是 0，直接退出了。
        
    - **正确做法**：始终在 `go` 关键字之前执行 `Add`。
        
- **禁忌：通过值传递 WaitGroup**
    
    - **后果**：Go 的参数传递是值拷贝。如果你把 `wg` 当作参数传给函数时没加指针（`*`），函数内部操作的将是一个**副本**，它的 `Done` 不会影响外部真正的计数器，导致 `Wait` 永远死锁。
        
    - **正确做法**：始终传递 `*sync.WaitGroup`（指针）。
        
- **禁忌：计数器变为负数**
    
    - **后果**：如果你调用 `Done()` 的次数超过了 `Add()` 的次数，程序会直接 **panic**。
        
    - **正确做法**：确保 `Add` 和 `Done` 是成对出现的。
# Chanel
## 什么是chanel
`channel` 是 Go 提供的并发通信机制，可以让多个 goroutine 安全地传递数据。
Channel 分为“无缓冲（Unbuffered）”和“有缓冲（Buffered）”两种。
#### 门派一：无缓冲通道 (Unbuffered Channel)

这是默认的 Channel 类型（`make(chan T)`）。它就像一场“必须面对面交接的快递”。

- **特点：** 同步。它没有任何内部空间来暂存数据。
    
- **阻塞规则：** * 当你向它**发送**数据时，发送方会**阻塞（卡住）**，直到有另一个 Goroutine 准备好接收这个数据。
    
    - 当你从它**接收**数据时，接收方会**阻塞**，直到有另一个 Goroutine 向它发送数据。
        
- **用途：** 保证两个 Goroutine 在某一时刻发生绝对的同步交接。
    

#### 门派二：有缓冲通道 (Buffered Channel)

创建时指定一个容量限制（`make(chan T, capacity)`）。它就像一个“快递驿站的快递柜”。

- **特点：** 异步（在容量范围内）。它有一个固定大小的队列。
    
- **阻塞规则：**
    
    - 只有当**缓冲区满了**，**发送**方才会阻塞。
        
    - 只有当**缓冲区空了**，**接收**方才会阻塞。
        
- **用途：** 允许发送方和接收方以不同的速度运行，解耦任务。
## 基本语法
```go
// 1. 创建 Channel
// 使用 make 函数创建，指定传递的数据类型（例如 int）
ch := make(chan int) 

// 2. 发送数据 (将 10 发送到通道 ch)
// 箭头指向 channel，表示数据流入
ch <- 10 

// 3. 接收数据 (从通道 ch 接收数据，并赋值给变量 x)
// 箭头从 channel 引出，表示数据流出
x := <-ch 

// 4. 关闭 Channel
// 告诉接收方：不会再有新数据发送进来了
close(ch)
```
### 死锁陷阱
如果你在同一个 Goroutine（例如主函数）里对一个无缓冲通道同时进行发送和接收，必然会导致死锁，整个程序会直接崩溃（Panic）。
**错误示例：**
```go
func main() {
    ch := make(chan int)
    ch <- 1 // 主协程在这里阻塞，等待有人接收...但永远等不到了，因为下面那行代码没机会执行
    fmt.Println(<-ch) 
}
```
正确做法：将发送或接收放到另一个 Goroutine 中。
```go
func main() {
    ch := make(chan int)
    
    go func() {
        ch <- 1 // 在后台 Goroutine 发送，阻塞等待交接
    }()
    
    fmt.Println(<-ch) // 主协程接收，交接成功，程序继续运行
}
```
# select
同时监听多个 channel，`select` 类似于 `switch`，但是它用于 channel 操作。
## 基本用法
```go
select {
case v := <-ch1:
	fmt.Println("从 ch1 收到", v)
case v := <-ch2:
	fmt.Println("从 ch2 收到", v)
}
```
如果多个 case 同时就绪，Go 会随机选择一个执行。如果都没有就绪，`select` 会阻塞。
**select 加 default**
```go
select {
case v := <-ch:
	fmt.Println(v)
default:
	fmt.Println("没有数据")
}
```
有 `default` 时，如果没有 channel 就绪，`select` 不会阻塞，而是直接执行 `default`。
# Mutex：互斥锁
虽然 Go 推荐使用 channel 通信，但有些场景使用锁更简单。比如多个 goroutine 同时修改一个变量。
## 不加锁的问题
```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	count := 0
	var wg sync.WaitGroup

	for i := 0; i < 1000; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()
			count++
		}()
	}

	wg.Wait()
	fmt.Println(count)
}
```
理论上输出为1000，但实际上输出不为1000，应为`count++`不为原子操作，而是分为三步，导致多个goroutine同时执行，发生数据竞争。
### 加锁的情况
```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	count := 0
	var wg sync.WaitGroup
	var mu sync.Mutex

	for i := 0; i < 1000; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()

			mu.Lock()//
			count++
			mu.Unlock()//
		}()
	}

	wg.Wait()
	fmt.Println(count)
}
```
# atomic：原子操作
如果只是简单计数，可以使用 `sync/atomic`。
```go
package main

import (
	"fmt"
	"sync"
	"sync/atomic"
)

func main() {
	var count int64
	var wg sync.WaitGroup

	for i := 0; i < 1000; i++ {
		wg.Add(1)

		go func() {
			defer wg.Done()
			atomic.AddInt64(&count, 1)//计数器
		}()
	}

	wg.Wait()

	fmt.Println(count)
}
```
适合 atomic 的场景：

|场景|是否适合|
|---|---|
|简单计数器|适合|
|状态标志位|适合|
|复杂结构体修改|不适合|
|多字段一致性更新|不适合|

复杂数据还是应该用锁或 channel。
# context
**Context（上下文）** 是 Go 语言并发模型中用来**控制 Goroutine 生命周期**的终极武器。

在实际开发中（尤其是 Web 服务后端），一个外部请求往往会衍生出成百上千个 Goroutine。如果这个外部请求被客户端突然取消了，或者因为超时失败了，那么它衍生出来的所有后台 Goroutine 还在傻傻地继续工作（比如继续查数据库、请求第三方接口），这会导致严重的**资源泄露**。
```
context常用于：
取消任务
超时控制
传递请求级别数据
控制 goroutine 退出
```
## 手动取消
```go
package main

import (
	"context"
	"fmt"
	"time"
)

func worker(ctx context.Context) {
	for {
		select {
		case <-ctx.Done():
			fmt.Println("worker 收到退出信号")
			return
		default:
			fmt.Println("worker working...")
			time.Sleep(500 * time.Millisecond)
		}
	}
}

func main() {
	ctx, cancel := context.WithCancel(context.Background())

	go worker(ctx)

	time.Sleep(2 * time.Second)

	cancel()

	time.Sleep(time.Second)
}
```
## 超时取消
```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()
```
```go
package main

import (
	"context"
	"fmt"
	"time"
)

func main() {
	ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
	defer cancel()

	select {
	case <-time.After(3 * time.Second):
		fmt.Println("任务完成")
	case <-ctx.Done():
		fmt.Println("任务超时:", ctx.Err())
	}
}
```
