## note
- 切片的三元组：cap，len，data（指针），注意cap的计算方法是从切片的起始位置到底层数组的末端
- 使用append在slice末尾添加数据时，若底层数组那个位置原来有数据会覆盖掉（不是往后挤）
- 切片扩容机制：若当前底层数组的容量不够，添加数据时切片会进行扩容，go发现原来的数组大小不够会另寻一块更大的内存进行创建数组存储（不是再原来数组的后面进行扩容），也就是**内存分离机制**
- Go 函数传参时，传进去的永远是实参的一个副本，只不过这个“副本”有时本身保存的是地址、指针或底层数据结构的引用信息。
# 创建方法
```go
// make([]类型, 长度len, 容量cap) cars := make([]int, 3, 5)
cars := make([]int, 3, 5)
cars := []int{10, 20, 30}

arr := [5]int{1, 2, 3, 4, 5} 
cars := arr[1:4] //包含头但不包含尾
```

# 增删改查
```go
cars := []int{10, 20, 30}

// 【查】: 
fmt.Println(cars[1]) // 输出: 20

// 【改】: 
cars[1] = 999
fmt.Println(cars)    // 输出: [10 999 30]
cars := []int{10, 20}
cars = append(cars, 30)       // 单个增加
cars = append(cars, 40, 50)   // 多个增加

//删：
cars := []int{10, 20, 30, 40}

// 目标索引：index = 1
// cars[:1] 是 [10] (拿前面的)
// cars[2:] 是 [30, 40] (拿后面的)
// 最后的 ... 叫做“打散操作符”，负责把后面的切片拆开，一个个塞进前面的切片里
cars = append(cars[:1], cars[2:]...) 

fmt.Println(cars) // 成功输出: [10 30 40]
```
